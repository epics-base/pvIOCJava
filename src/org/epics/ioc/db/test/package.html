EPICS JavaIOC db test
package: org.epics.ioc.db.test
2007.01.09

Introduction

This package is an eclipse JUnit test for org.epics.ioc.dbAccess. It contains The following JUnit tests:

  • XMLToDataBaseTest
    Reads a set of database definition files and a set of record instance files. It dumps the record instances.
  • XMLAllTypesTest
    Reads a set of definitions that include all types.
  • DefaultTest
    Reads a set of database definition files that have defaults for all fields that are allowed to have defualts. Creates an instance definition which does not set any field values. It dumps the resultig instance.
  • ParentTest
    Reads a set of database definitions and record instances. Show the parent and record for specified fields.
  • AccessTest
    Tests DBAccess.
  • ListenerTest
    Tests DBListener.
  • ReplaceTest
    Test repacing the default implementation of all the field types.
  • SubstituteTest
    Tests include and macro substitution. This also is a test for pv, dbd, and db because XMLToDBD is called, which makes extensive use of dbd and pv, and XMLToIOCDB is called, which makes extensive use of db. It also provides an example of parsing database definitions.

It also has a Java class TestListener, which is used by some of the other tests and is an example of implementing a DBListener.

The examples use the DBD definitions defined in javaIOC/dbd.

The following database definition files are available:

  • accessDBD.xml
    Include statements for the files requires by AccessTest.
  • aiDBD.xml aiRaw, aiLinear, aiLinearRecord
  • aiLinkedDBD.xml
    Definitions for an ai record that links to an aiRaw record.
  • allTypesDBD.xml
    A structure and recordType that has fields of every supported type.
  • analogDBD.xml
    A simple recordType definition.
  • defaultDBD.xml
    Like allTypesDBD.xml but assigns defaults to all fields that are allowed to have defaults.
  • listenerDBD.xml
    Include statements for the files requires by ListenerTest.
  • parentDBD.xml
    Include statements for the files requires by ParentTest.
  • powerSupplyDBD.xml
    A simple power supply for power, current, and voltage.
  • powerSupplyLinkedDBD.xml
    A simple power supply that links to other records for current and voltage.
  • replaceDBD.xml
    Include statements for the files requires by ReplaceTest.
  • substituteDBD.xml
    Include statements for the files requires by SubstituteTest.
  • xmlToDataBaseDBD.xml
    Include statements for the files requires by XMLToDataBaseTest.

The DB.xml files are read by the various tests. These database files and tests are good examples of the kinds of data a JavaIOC can hold.
